# What `configme install yelmox` does

This page is for context only. The recommended way to install YelmoX is to run
`configme install yelmox` (see the [Quick start](index.md#quick-start)); you do
**not** need to run any of the commands below by hand. They are reproduced here
so you can see exactly what `configme` clones, configures, links, and builds on
your behalf.

`configme install` performs the following phases:

1. **Clone** the YelmoX checkout and each component repository it needs
   (`fesm-utils`, `yelmo`, `FastIsostasy`, `rembo1`).
2. **Manifest** — write `.configme/manifest.toml` so the checkout is
   self-describing regardless of its directory name.
3. **Link** each component into the directories where the build expects it.
4. **Configure** — generate a system-specific `Makefile` for every package from
   shared machine/compiler fragments, with the netCDF paths detected
   automatically.
5. **Build** `fesm-utils` (LIS + FFTW + utils). This is the slow step
   (~10-30 min), which is why it is folded into the install rather than run
   separately.
6. **Extras** — install `runme` if it is missing, create `.runme_config`, and
   link the external data paths (`ice_data`, `isostasy_data`).

Each run also writes the exact commands it executed to a `.install.sh` script in
the checkout, so any install is reproducible. The script below is the
equivalent for the `dkrz_levante` machine with the `ifx` compiler; substitute
your own machine and compiler, and note that the netCDF paths (`NC_FROOT` /
`NC_CROOT`) and the install root are filled in automatically for your system.

```bash
#!/usr/bin/env bash
# Generated by `configme install` — reproduces this install.
set -euo pipefail
# target=yelmox  machine=dkrz_levante  compiler=ifx

# --- clone ---
git clone git@github.com:fesmc/yelmox.git $YELMOXROOT
git clone git@github.com:fesmc/fesm-utils.git $YELMOXROOT/fesm-utils
git clone git@github.com:fesmc/yelmo.git $YELMOXROOT/yelmo
git clone git@github.com:palma-ice/FastIsostasy.git $YELMOXROOT/FastIsostasy
git clone git@github.com:alex-robinson/rembo1.git $YELMOXROOT/rembo1

# --- manifest ---
mkdir -p $YELMOXROOT/.configme  # write .configme/manifest.toml

# --- links ---
ln -s ../fesm-utils $YELMOXROOT/yelmo/fesm-utils
ln -s ../fesm-utils $YELMOXROOT/FastIsostasy/fesm-utils
ln -s ../fesm-utils $YELMOXROOT/rembo1/fesm-utils

# --- configure (per package) ---
# fesm-utils: build with autotools (slow, ~10-30 min):
(cd $YELMOXROOT/fesm-utils && NC_FROOT=<auto-detected> NC_CROOT=<auto-detected> ./build.py --variant both -m dkrz_levante -c ifx)
(cd $YELMOXROOT/fesm-utils/utils && configme config fesm-utils/utils -m dkrz_levante -c ifx)
# fesm-utils/utils: build (serial, omp):
(cd $YELMOXROOT/fesm-utils/utils && make openmp=0 fesmutils-static)
(cd $YELMOXROOT/fesm-utils/utils && make openmp=1 fesmutils-static)
(cd $YELMOXROOT/yelmo && configme config yelmo -m dkrz_levante -c ifx)
(cd $YELMOXROOT/FastIsostasy && configme config FastIsostasy -m dkrz_levante -c ifx)
(cd $YELMOXROOT/rembo1 && configme config rembo1 -m dkrz_levante -c ifx)
(cd $YELMOXROOT && configme config yelmox -m dkrz_levante -c ifx)

# --- extras ---
command -v runme >/dev/null 2>&1 || pip install git+https://github.com/fesmc/runme
# runme_config: create .runme_config and set hpc/account
runme --config  # or: cp .runme/runme_config .runme_config
# data_link ice_data: ln -s <path> ice_data
# data_link isostasy_data: ln -s <path> isostasy_data
```
